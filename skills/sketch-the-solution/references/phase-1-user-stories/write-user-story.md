# UX Step 1.2 — Write User Story

Output "Read UX Write User Story skill." to chat to acknowledge you read this file.

Phase: Phase 1 — User Stories → Step 2 of 3

Write narrative-form user stories that describe each avatar's CURRENT challenge end-to-end. Stories describe pain, not your solution.

## Process

1. **For each avatar/goal pair**, write a story that:
   - Starts where the challenge begins (or even before)
   - Narrates every step they take to accomplish their goal
   - Includes motivations, emotions, environment, reasoning
   - Covers the frustrations and friction at each step
   - Ends at task completion (or failure)

2. **Write from multiple perspectives.** If a user posts something and another user responds, write BOTH sides.

3. **Dig for depth** — find "the need behind the need":
   - Why do they use this particular tool/method?
   - Why does that matter to them?
   - What's the deeper motivation?

4. **Generate both formats:**

**Narrative format:**
```markdown
### [Avatar Name]'s Story: [Goal]

[Full narrative paragraph describing current-state pain, step by step]
```

**Formal template:**
```markdown
As a [user role], I want to [verb-centric behavior], so that [user value added].
```

## Rules

- Stories describe the CURRENT challenge — NOT how your product solves it
- Include maximum detail: motivations, emotions, environment, reasoning
- Write stories for EVERY avatar, not just the primary user
- These are living documents — update after future customer calls
- "Failure is encouraged at this stage — neither costs you anything"

## Output

Append to `user-stories.md`: User Stories section with narrative + formal template per avatar/goal.
