# UX Step 5.2 — Inform → Engage → Invite

Output "Read UX Inform Engage Invite skill." to chat to acknowledge you read this file.

Phase: Phase 5 — Screen Requirements → Step 2 of 3

Structure each screen's intent using the three-part framework. Every screen should Inform the user (build trust/context), Engage them (show value), and Invite them forward (call to action).

## Process

1. **For each screen**, define three layers:

```markdown
### [Screen Name]

**Inform** — What context/trust does the user need?
- [e.g., "Benefits per user type", "Testimonials", "Press mentions"]
- [e.g., "Breadcrumb showing location in the app"]

**Engage** — What value does the user get here?
- [e.g., "Preview of member profiles with relevant skills"]
- [e.g., "Content excerpts with author and date"]

**Invite** — What action should the user take next?
- [e.g., "View Full Profile button", "Post Content CTA"]
- [e.g., "Search/filter controls to refine results"]
```

2. **Landing pages get extra attention:**
   - **Inform:** headline, sub-copy, how-it-works steps, trust elements (featured companies, testimonials, press)
   - **Engage:** show value before asking for anything — previews, demos, examples
   - **Invite:** primary CTA per user type, secondary CTA (learn more)

3. **Internal screens are simpler:**
   - **Inform:** breadcrumb, page title, current state
   - **Engage:** the content/data/functionality
   - **Invite:** primary action button, navigation to related screens

## Rules

- Engage before asking for anything — show value first, then request credentials
- Trust elements are critical on entry/landing screens
- Every screen must have at least one clear Invite (CTA)
- The Inform layer prevents users from feeling lost
- Apply the Instant Gratification principle from Phase 3

## Output

Append to `screen-requirements.md`: Per-screen Inform/Engage/Invite breakdown.
