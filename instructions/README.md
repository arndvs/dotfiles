# Instructions

Context, service, and task-triggered knowledge packs (Tiers 1–2). Each file loads based on environment detection, active services, or current task type.

## How Instructions Load

`CLAUDE.md` (generated by bootstrap from `CLAUDE.base.md`) references instructions via `@`-includes. Loading is tiered:

- **Always loaded:** Referenced unconditionally in `CLAUDE.base.md`
- **Context-triggered:** Loaded when `$ACTIVE_CONTEXTS` (set by `detect-context.sh`) matches the instruction's target stack
- **Service-triggered:** Loaded when a specific external service is active or mentioned
- **Task-triggered:** Loaded when the current task type matches

## Private Instructions

`instructions/_local/` is gitignored. Files here are auto-discovered by bootstrap:

- Files **without** `auto-load: false` in frontmatter → appended as always-loaded `@`-references in `CLAUDE.md`
- Files **with** `auto-load: false` → registered as task-triggered (loaded only when task matches the description)

## Adding an Instruction

1. Create `instructions/your-stack.instructions.md`
2. Reference it in `CLAUDE.base.md` under the appropriate loading section (always, context, service, or task)
3. Run `ctrl bootstrap` to regenerate `CLAUDE.md`

For private instructions, drop into `instructions/_local/` — bootstrap auto-appends the `@`-reference.
